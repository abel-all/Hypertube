    Foreword
    

    

    

    Preliminary instructions
    

    

    Please confirm with the students the libraries they used for the project.
    
Please note that libraries allowing the direct streaming of videos from
    
torrents are not permitted.
    
Therefore, the use of libraries such as webtorrent, pulsar, or peerflix 
    
will be considered cheating.
    


    Cheating results in termination of the evaluation process.
    


    Any credentials, API keys or environment variables must be stored in a .env
    
file during the evaluation.
    
If any credentials or API keys are found in git repository or outside of 
    
the .env file, the evaluation will be stopped and the mark will be 0.
    


    Moreover, if any warning, error or notices appear in the web console
    
during the evaluation, select Crash on top of the grading system. An error
    
message from 500 to 599 is also considered a Crash.
    


    Important: In order to comply with copyright legislation and avoid any
    
illegal torrenting activity, you must only use external sources that 
    
provide royalty-free or legally distributable content.
    


    Examples of allowed sources include:
    


      

    • http://www.legittorrents.info
      • 

    • https://archive.org
      • 

    


    You are responsible for ensuring that the sources you use comply with the 
    
copyright laws and regulations applicable in your country. Any download or 
    
stream of copyrighted material without proper rights is strictly prohibited.
    


    

    

    

    

    





    

    

    Security
    

    

    The subject was clear, the site must be secured.
    
Make sure that:
    


      

    • Passwords are encrypted in the DB.
      • 

    • Data entry and upload forms have correct validations.
      • 

    • SQL injection is impossible.
      • 

    


    If any of these conditions have not been met, the evaluation stop.
    


    

    

    

    

    





    

    

    

    

    

    

    Users
    

    

    

    Creation and management of user account
    

    

    Users should be able to subscribe with at least these pieces of information:
    


      

    • Email address
      • 

    • User name
      • 

    • Profile picture
      • 

    • First and last name
      • 

    • Password (secured; any commonly used English word will not be accepted)
      • 

    


    These pieces information should be editable by the user who is currently
    
logged in.
    


    

    

    

    

    





    

    

    User registration
    

    

    Registration and login must be possible with an Omniauth.
    
At least two strategies are required: the 42 strategy and one chosen
    
by the evaluated student.
    


    User must be able to log in with their ID and should be able to receive
    
an email to reset their password if they forget it.
    


    Users should be able to log out from any page on the website.
    


    

    

    

    

    





    

    

    

    

    

    

    Library
    

    

    

    Access
    

    

    Is the library accessible only to logged-in users?
    


    

    

    

    

    





    

    

    Default layout
    

    

    By default, if no search is being conducted, the library must display thumbnails
    
of the most popular media from the sources managed by the application.
    
The list must be sorted based on a condition set by the evaluated student.
    


    

    

    

    

    





    

    

    Miniatures
    

    

    The thumbnails should contain the name of the video, and if available,
    
the year of production, rating and a cover image.
    


    All of this information should be available for the most popular media.
    


    Additionally, viewed videos must be clearly differentiated from the non-viewed ones.
    


    

    

    

    

    





    

    

    Pagination and sorting out
    

    

    The list of videos must be paginated, and the pages must load asynchronously.
    


    Users should also be able to sort the list according to different criteria.
    


    

    

    

    

    





    

    

    Search
    

    

    The search engine should query at least two different external sources
    
and limit the results to videos.
    


    The result of a search must be displayed as thumbnails sorted by
    
names.
    


    As an example, you can search for a video using the following command:
    
curl http://www.randomlists.com/random-movies\?qty\=1 | cat | ruby -e "p gets.match(/port'>([^<]*)/)[1]".
    


    

    

    

    

    





    

    

    

    

    

    

    Video part
    

    

    

    Access
    

    

    Is the section containing the player only available to logged-in users?
    


    

    

    

    

    





    

    

    Informations about the video
    

    

    The video section should also display information about the video being
    
played, such as an abstract, the cast, production year, and other relevant details.
    
Popular videos should be tested to ensure these details are properly displayed.
    


    Users should also find a list of comments left by other users and leave their 
    
own comments.
    


    

    

    

    

    





    

    

    Downloading a video
    

    

    When a user launches a video, the server should use the BitTorrent protocol
    
to download the video. Once the entire video is downloaded and can run without issue,
    
the player should begin streaming it. 
    


    The download process should be a background, non-blocking activity.
    


    

    

    

    

    





    

    

    Video conversion
    

    

    When the video cannot be played natively by the browser, it should
    
be converted on-the-fly into a format that can be streamed. 
    
The system should support at least the mkv format.
    


    The following magnet should be streamable, for example: magnet:?xt=urn:bt
    
ih:79816060ea56d56f2a2148cd45705511079f9bca&dn=TPB.AFK.2013.720p
    
.h264-SimonKlose&tr=udp%3A%2F%2Ftracker.openbittorrent.com%3A80&
    
tr=udp%3A%2F%2Fopen.demonii.com%3A1337&tr=udp%3A%2F%2Ftracker.co
    
ppersurfer.tk%3A6969&tr=udp%3A%2F%2Fexodus.desync.com%3A6969
    


    

    

    

    

    





    

    

    Saving videos
    

    

    A video that has been downloaded (by any user) must be saved on the
    
server and deleted if it has not been viewed in the following month.
    
Of course, playing a video on the server should not trigger a new download;
    
It should stream from the copy on the server.
    


    

    

    

    

    





    

    

    Subtitles
    

    

    If English subtitles are available for the video, they must be
    
automatically downloaded and selectable on the player.
    


    If the language of the video is different from the user's preferred language,
    
and subtitles are available in the preferred language, they should be automatically
    
downloaded and made selectable on the player.

    Basic API checks
    

    

      

    • Is there an RESTful API available? Can you authenticate with a client ID and secret?

    Users
    

    

    The user's API calls should work and return the expected data, including 
    
the username, email and profile picture URL.
    
Additionally, the PATCH method should allow a user to modify their information.
    


    

    

    

    

    





    

    

    Movies & comments
    

    

    GET calls for movies and comments are working properly and returning 
    
the expected data.
    
You are also able to post new comments and update existing comments
    
using the provided routes.
    


    

    

    

    

    





    

    

    Disabled calls
    

    

    HTTP error code are returned for calls that are not listed in the documentation.
